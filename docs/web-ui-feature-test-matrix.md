# Web UI Feature-Test Matrix

This document expands F32 in `docs/feature-test-management.md` into smaller
Web UI requirements. A Web UI change is not considered protected unless the
matching row below has at least one requirement-level test and, when relevant,
a host/runtime integration test.

| Requirement | Required behavior | Test evidence |
|---|---|---|
| Authenticated local host | Web host binds locally, serves embedded assets, uses a per-process token, and does not leak token/referrer data. | `parses_basic_web_launch_options`, `rejects_ports_outside_the_local_web_range`, `generated_local_access_token_has_expected_entropy_shape`, `embedded_frontend_assets_receive_browser_safe_content_types`, `browser_responses_disable_referrer_leaks_and_remote_active_content`, `removes the access token from the visible URL while retaining the session credential`. |
| Session creation and naming | Users can create sessions, default names are `SessionN`, and user renames are reflected without changing worker identity. | `session_create_returns_the_complete_session_to_the_requesting_browser`, `unnamed_web_session_uses_session_name_while_worker_keeps_core_identity`, `uses session terminology consistently for the creation workflow`, `supports agent rename and a distinct animated working state`. |
| Per-session runtime profile | Each Web Session can carry provider/model/protocol/env overrides without mutating defaults or leaking API keys. | `session_creation_applies_independent_runtime_env_without_mutating_defaults`, `runtime_update_refreshes_new_session_defaults_without_rewriting_existing_sessions`, `session_runtime_env_rejects_unknown_empty_and_invalid_values`, `creates sessions with independent runtime environment overrides`. |
| Multi-session topic isolation | Core topics, worker activity, final answers, decisions, and cwd updates route only to the owning Session and known Context/Worker. | `concurrent_agent_topics_stay_in_their_own_session_and_wire_payload`, `real_concurrent_workers_route_final_topics_to_matching_web_sessions`, `mismatched_core_topic_is_not_forwarded_or_written_to_another_agent`, `applies a model response only to the session named by the core topic`, `does not append a core topic event to another session with the same turn id`, `does not append scoped core topics for unknown workers or contexts`, `applies a structured cwd update only to the matching session`, `rejects core topics scoped to an unknown context before mutating a session`. |
| Worker hierarchy and state | The UI can show primary/child workers while child worker details remain scoped to the owning Session turn, including lifecycle topics that introduce a new child Context/Worker. | `one_session_aggregates_primary_and_subworker_state_without_cross_finishing`, `child_context_worker_uses_its_owning_sessions_runtime_profile_and_env`, `accepts lifecycle topics that introduce a new scoped worker and context`, `expands each session into its scoped worker status list`, `keeps a matched agent working without changing unrelated sessions`. |
| Stop/cancel under human pressure | Repeated Stop clicks are idempotent, repeated Send clicks cannot duplicate one draft, pending sends do not erase text typed during the send, draft text and pending send guards are isolated per Session, stale drafts/pending locks are pruned when snapshots swap out sessions, cancel stops all workers in the Session, stale Stop is harmless, and the next turn uses only the primary worker. | `cancel_stops_all_session_workers_and_next_turn_runs_only_primary`, `duplicate_cancel_commands_are_idempotent_for_one_active_turn`, `uses synchronous pending guards for rapid repeated browser clicks`, `guards one browser draft submission while preserving text typed during the pending send`, `keeps drafts and pending send guards isolated by session`, `prunes stale drafts and pending send locks when a snapshot swaps out sessions`, `keeps draft text and releases the pending guard when cancellation blocks a reserved send`, `keeps a human click storm bounded and session scoped`, `binds assistant-ui running state to the authoritative session lifecycle`. |
| Send during active work | While the model is working, Send is a supplement to the active turn, not a no-op and not a new task. Rapid repeated sends preserve order. | `sends working-session text as a supplement instead of disabling input`, `rapid_submit_during_an_active_turn_is_treated_as_a_supplement`, `repeated_user_sends_during_an_active_turn_are_ordered_supplements`, `user_supplement_is_retained_in_the_authoritative_web_session_snapshot`. |
| Stale supplement recovery | A supplement arriving after a cancel/completion race becomes a new task instead of failing with `active_turn_not_found`. | `stale_supplement_after_cancel_completion_starts_a_new_turn`, `stale_supplement_after_cancel_consumes_pending_attachments_as_a_new_task`, `stale_topic_reply_after_turn_completion_is_ignored_without_host_error`, `sends a new task after a cancelled active turn is marked finished`. |
| Attachments | Users can upload, remove, and submit files. Submitted files move into the correct user entry, long names remain inspectable, and failed/racy paths do not drop files. | `pending_upload_moves_into_the_submitted_user_entry_and_is_not_reinjected`, `pending_attachment_removal_is_session_scoped_and_deletes_the_stored_file`, `duplicate_pending_attachment_removal_is_idempotent_for_the_same_session`, `failed_pending_attachment_file_removal_restores_the_session_entry`, `active_turn_supplement_consumes_pending_attachments_into_the_same_turn`, `failed_active_turn_supplement_does_not_drop_pending_attachments`, `lets users remove pending attachments without losing access to long file names`, `moves submitted files from the composer into a compact user attachment list`. |
| Inline decisions | Core/worker requests appear in the owning turn flow and replies route to the requesting worker without cross-session overwrite. | `child_context_worker_uses_its_owning_sessions_runtime_profile_and_env`, `stale_work_instruction_reply_during_new_active_turn_is_ignored`, `stale_topic_reply_after_turn_completion_is_ignored_without_host_error`, `queues concurrent decisions by session and request id without cross-session replacement`, `queues concurrent decisions from different workers in the same session`, `uses the shared worker-aware decision key for inline request pending state`, `clears only the resumed workers decision within a shared session`, `uses an explicit session-created event and session-scoped inline decisions`. |
| Work instructions | AGENTS/CLAUDE decisions are not announced before acceptance, use timeout/default behavior, and stale replies are ignored. | `ask_mode_does_not_announce_work_instructions_before_user_acceptance`, `ask_mode_queues_the_first_turn_then_loads_work_instructions_after_matching_reply`, `ask_mode_decline_continues_the_turn_without_loading_work_instructions`, `ask_mode_rejects_a_mismatched_reply_without_releasing_the_pending_turn`, `stale_work_instruction_reply_during_new_active_turn_is_ignored`, `does not turn work-instruction bookkeeping into user-visible activity`. |
| Current cwd display | Session list and composer show the current cwd, and successful `self_tool chg_cwd` updates only after core reports the new cwd. | `successful_cwd_action_updates_only_its_session_and_reconnect_snapshot`, `real_worker_cwd_tool_call_updates_web_session_state`, `shows the live session cwd in navigation and above the composer`, `shows the tail of a long cwd while retaining short paths verbatim`. |
| Turn process rendering | Free talk, progress, actions, repairs, context compact, and tool rows render inside the task process frame without noisy invented labels or stale running rows. | `uses frame styling without repeating user or session identity labels`, `groups each task into user input, bounded process, and separate final delivery`, `coalesces tool lifecycles and renders tools as compact subordinate rows`, `replaces an action start with its terminal lifecycle event`, `pairs duplicate concurrent actions in order without collapsing either invocation`, `pairs action lifecycle events even when input object key order changes`, `pairs action lifecycle events when nested input object key order changes`, `keeps context compaction as a typed system activity with token metrics`, `renders context compaction outside chat messages with a reduced-motion fallback`. |
| Final answer rendering | Final answers render as Markdown with code highlighting, tables, task lists, safe links, and completion telemetry below the answer. | `renders GFM and highlighted code with a copy affordance`, `uses the Markdown highlighter for final answers and Bash activity commands`, `attaches completion telemetry only to the matching final answer`, `turn_completion_stats_are_attached_to_the_matching_final_answer`, `renders live task usage and session context without replacing final telemetry`. |
| Usage and context status | While a turn is running and after it finishes, token/cached/context stats stay attached to the correct Session and turn. | `live_model_usage_is_retained_in_the_active_turn_and_correct_session`, `aggregates live task usage across model rounds and preserves the latest call`, `uses only the selected session's latest real provider usage for context`, `turn_completion_stats_are_attached_to_the_matching_final_answer`. |
| History and resume | Web restores recent chat, loads older history by pages, tolerates malformed records, and preserves task/supplement/approval semantics. | `stored_session_restores_after_web_host_restart_with_fresh_worker`, `history_page_command_loads_older_records_by_cursor`, `history_page_command_skips_malformed_records_without_breaking_cursor`, `restored_web_turns_follow_history_time_not_turn_id_lexical_order`, `restored_web_turns_preserve_user_entry_kinds`, `turn_user_entries_are_persisted_with_raw_text_and_semantic_kind`, `restores task, supplement, and approval user entries inside one turn`. |
| Mem switching | Users can see/switch mem spaces; switching swaps Sessions to the selected mem, rejects path traversal, selects a live Session from the new snapshot, does not carry stale browser draft or pending-send state into the newly loaded Session set, and locks old-session send/upload/remove/history/decision/create/cancel/rename actions while the switch is pending. | `snapshot_reports_the_active_mem_space_and_paths`, `mem_switch_swaps_out_sessions_and_loads_the_selected_space`, `mem_switch_rejects_paths_and_parent_traversal`, `moves the active session to a live session when a snapshot swaps out the old one`, `moves active selection to a live session when a reconnect or mem snapshot swaps sessions`, `prunes stale drafts and pending send locks when a snapshot swaps out sessions`, `recovers from an in-flight old-mem send after a mem snapshot swaps sessions`, `does not send new tasks or supplements while a mem switch is pending`, `does not rename a session while mem switching or another rename is pending`, `locks old-session interactions while a mem switch snapshot is pending`. |
| Appearance | Dark/light/font/text-size settings persist, tolerate malformed storage, and remain separate from core behavior. | `appearance.test.ts`, `persists theme, font, and text-size appearance without changing core state`. |
| Scroll and bounded rendering | Long conversations, reconnect snapshots, and high-frequency event bursts remain bounded, preserve scroll position, and avoid horizontal overflow. | `scroll.test.ts`, `bounds a reconnect snapshot with many turns and high-frequency events`, `keeps repeated live event bursts bounded and isolated across sessions`, `keeps a human click storm bounded and session scoped`, `bounds newly appended turns without changing chronological order`, `active_turn_event_windows_are_bounded_and_session_isolated`, `active_turn_user_entries_drop_only_the_oldest_entries_at_the_bound`, browser smoke rows in `docs/manual-release-smoke.md`. |
| Diagnostics and host errors | Debug activity is optional by default, but host/session errors remain visible outside the diagnostic panel. | `shows host and session errors outside the default-hidden diagnostic panel`, `defaults the diagnostic activity panel to hidden`, `host_error` handling in frontend contract tests. |
| Release packaging | The Web bundle is reproducible, embedded in the Rust host, and dependency licenses are allowed. | frontend TypeScript/Vite build in `scripts/ci.sh`, `scripts/web_license_check.sh`, `cargo build --locked -p timem_shell -p timem_web --release`, Linux/macOS GitHub CI. |

When a Web UI feature changes, update this matrix in the same commit as the
code and tests. If a row cannot be fully automated, record the manual smoke in
`docs/manual-release-smoke.md`.
